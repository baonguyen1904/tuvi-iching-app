    **Implementation notes:**
    - Load laso_points.json once at module import: `_POINTS: dict = json.load(open(Path(__file__).parent.parent / "data" / "laso_points.json"))`
    - Use `unicodedata.normalize("NFC", sao)` when looking up sao names (HTML source may be NFD)

    - Positive points → duong accumulator; negative points → am accumulator
    - Dimension scores (the separate per-dimension weight columns) are ADDITIVE multipliers: when a sao has non-null su_nghiep value, add that value to su_nghiep total separately

    **Wait** — the actual scoring logic: reading the xlsx, each sao has:
    - `point`: the base point (pos=Dương contribution, neg=Âm contribution)
    - `su_nghiep`, `tien_bac`, etc.: per-dimension weight (how much that sao contributes to that specific dimension)

    So for dimension scoring:
    - For each sao in a cung, if `dimensions["su_nghiep"]` is not None, add it to su_nghiep total
    - If the dimension value is positive → adds to dimension_duong
    - If negative → adds to dimension_am
    - If zero or positive under a threshold → could be neutral (TB)

    For the MVP, implement it as: dimension_score = sum of all non-null dimension[dim] values for sao in cung.
    Return as {"duong": sum_of_positives, "am": sum_of_negatives, "tb": 0} (TB scoring can be refined later).
