# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

Phase 1 frontend is intentionally minimal (source: CONTEXT.md D-19). The only UI deliverable is the birth data input form and its companion states. Landing page, result page, processing screen, charts, and dimension buttons are Phase 3 scope — do not implement them here.

**Phase 1 UI surface:**
- Birth data input form (4 fields: date, canh gio, gender, name)
- "Khong ro" disclaimer banner below the form when that option is selected
- Submit button with loading state
- Success state: la so confirmed, shows raw confirmation (not full result — that is Phase 3)
- Error state: cohoc.net unavailable, Vietnamese message + retry button

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.2 |

**Font family:** `Inter, "Noto Sans", sans-serif` — Inter covers Vietnamese diacritics well; Noto Sans is the fallback for glyphs Inter may miss.

**Note:** Display size (28–32px) is reserved for Phase 3 (landing page hero, result page header). Do not introduce it in Phase 1.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F14` (near-black, deep indigo-black) | Page background, form page surface |
| Secondary (30%) | `#1C1C2E` (dark navy) | Form card background, input backgrounds |
| Accent (10%) | `#C9A84C` (warm gold) | Submit button, active state for selected radio/dropdown, focus ring |
| Destructive | `#DC2626` (red-600) | Error message text and border, retry state |

**Text colors:**
- Primary text: `#F0EDE8` (off-white, warm) — body copy, input values
- Secondary text: `#9CA3AF` (gray-400) — placeholder text, helper copy, disclaimer
- Label text: `#D1C9BC` (warm light gray) — field labels

Accent reserved for: submit button background, selected radio option border + fill, dropdown focused border, focus rings on interactive elements.

**Rationale:** A Vietnamese Tu Vi astrology app warrants a deep, authoritative color palette (dark background, gold accent) rather than a clinical tech palette. This palette is established in Phase 1 and carries through all phases. If the developer prefers a light-mode palette instead, substitute `#FAFAF7` (dominant), `#F0EDE8` (secondary), `#7C5A1E` (accent dark gold) — but this must be a deliberate decision made before Phase 1 implementation begins, as it becomes the cross-phase contract.

## Form Field Specification

### Date field (DINP-01)
- Label: **"Ngay sinh (Duong lich)"**
- Component: DatePicker with calendar popover
- Validation: Required. Must be a valid past date.
- Error copy: "Vui long chon ngay sinh."

### Canh gio field (DINP-02)
- Label: **"Gio sinh"**
- Component: Select dropdown
- Options (in order): Khong ro, Ty (23:00–01:00), Suu (01:00–03:00), Dan (03:00–05:00), Mao (05:00–07:00), Thin (07:00–09:00), Ty (09:00–11:00), Ngo (11:00–13:00), Mui (13:00–15:00), Than (15:00–17:00), Dau (17:00–19:00), Tuat (19:00–21:00), Hoi (21:00–23:00)
- Default: "Khong ro" pre-selected
- Disclaimer trigger: selecting "Khong ro" shows the disclaimer alert

### Gender field (DINP-03)
- Label: **"Gioi tinh"**
- Component: RadioGroup, horizontal layout on desktop, stacked on mobile
- Options: "Nam" | "Nu"
- Validation: Required. No default — user must choose.
- Error copy: "Vui long chon gioi tinh."

### Name field (DINP-04)
- Label: **"Ho va ten"** with "(tuy chon)" in secondary color inline
- Component: Input text field
- Placeholder: "Nguyen Van A"
- Validation: Optional; no error state

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (submit button) | "Lap la so" |
| Submit loading state | "Dang xu ly..." |
| Disclaimer (unknown time) | "Ban chon 'Khong ro' gio sinh. He thong se su dung gio mac dinh Ty (23:00–01:00). Ket qua tinh toan phu thuoc gio sinh co the khong chinh xac." |
| Empty state heading | Not applicable in Phase 1 (no list/data views) |
| Success confirmation | "La so da duoc tao thanh cong." (shown as transient confirmation before Phase 3 result page) |
| Error state heading | "Khong the ket noi den he thong." |
| Error state body | "Co loi xay ra khi lay la so. Vui long thu lai sau vai giay." |
| Retry button label | "Thu lai" |
| Destructive actions | None in Phase 1 |

**Tone requirement (non-negotiable from CLAUDE.md):** All copy must be positive and empowering. Error messages describe the problem and offer a clear next step — no blame language ("loi cua ban", "sai dinh dang").

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Default form | Page load | All fields empty; submit button enabled (but form invalid until date + gender filled) |
| "Khong ro" selected | User selects Khong ro in canh gio | Disclaimer alert appears below form with fade-in |
| Validation error | User submits with missing required fields | Required fields show red border + error copy below; no toast |
| Loading | Submit clicked, form valid | Submit button shows spinner + "Dang xu ly..."; button disabled; form inputs disabled |
| Success | Backend returns la so confirmed | Brief success message; Phase 3 handles navigation to result page |
| Error / retry | Backend call fails (cohoc.net unavailable) | Error alert shown above submit button; "Thu lai" button replaces submit |

---

## Layout

- Form is centered on page, max-width 480px on desktop, full-width on mobile with 16px horizontal padding
- Form card uses secondary color background with 24px padding and 8px border-radius
- Fields stacked vertically with 16px gap between each field group (label + input)
- Submit button is full-width, 48px tall
